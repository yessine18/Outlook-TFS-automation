Chapter 3 — Iteration 1

1

Basic API Integration (Email → Azure DevOps)

Technologies Introduced

  • .NET 10 Worker Service (BackgroundService)
  • Microsoft Graph SDK v5
  • Azure DevOps REST API
  • Azure AD ClientSecretCredential

What It Does

Polls the shared Outlook mailbox every 60 seconds. For each unread email, creates an Azure DevOps Issue with the raw subject and body, then marks the email as read.

📨 Outlook Inbox
↓ Graph API
🔄 .NET Worker (poll every 60s)
↓ REST API
📋 Azure DevOps Issue

⚠️ Limitation: "Garbage In"

Raw HTML dumped into ADO. No extraction, no classification, no routing. Every ticket goes to the default assignee.

Chapter 3 — Innovation Highlight

Model Context Protocol (MCP) Architecture

What is MCP?

Open-source standard by Anthropic providing a secure, unified way for LLMs to connect to external data sources — separating AI reasoning from API authentication.

🏆 Why We Built a Custom MCP Server

Microsoft's official MCP Server only fetches documentation. It does not support training modules, learning paths, or certification exams.

We built learn_catalog_mcp.py to bridge this gap — effectively implementing the feature Microsoft hasn't released yet.

Security Architecture

Design Decisions

Database-First Persistence: Ticket saved to PostgreSQL before creating the ADO work item. If ADO fails, data is safe.

PipelineStatus as String: Stored via .HasConversion<string>() for human readability in raw SQL queries.

Scoped DbContext: IServiceScopeFactory creates fresh contexts inside the Singleton background service.

Chapter 3 — Reliability

Error Handling & Graceful Degradation

flowchart TD A["Pipeline Step Fails"] --> B{"Which step?"} B -->|LLM Analysis| C["❌ ABORT\nMark as read"] B -->|Database| D["❌ ABORT\nMark as read"] B -->|ADO Creation| E["DB: AdoFailed"] B -->|Email Sending| F["DB: MailSendFailed"] B -->|RAG Evaluation| G["⚠️ Continue\nwithout RAG"] C --> H["📧 TMA Alert Email"] D --> H E --> H F --> H G --> I["Pipeline continues\nnormally"]

Resilience Patterns

Step-Level Isolation

Each pipeline step (LLM → DB → ADO → Email) wrapped in independent try/catch blocks. Downstream steps survive upstream failures.

Graceful RAG Degradation

If the Agentic RAG pipeline fails, the system continues normally — ticket is created for human assignment. No crash.

TMA Alert System

Formatted HTML error reports sent to Application Support team with: failed step, email subject, sender, error type, and UTC timestamp.
